Part C

How to set me up

For a teammate cloning the project from GitHub. Tick each step as you go.

  • Install Claude Code and sign in.
  • Clone the repo and open the project folder.
  • Confirm the layout: CLAUDE.md, .claude/agents/architect.md, .claude/agents/qa.md, skills/skills-first/SKILL.md.
  • Check paths: I must resolve skills/skills-first/SKILL.md and its references/ folder.
  • First run: launch Claude Code in the folder and send me the trigger message.
git clone <your-repo-url> skills-first-agent
cd skills-first-agent
claude   # start Claude Code in this folder
Part C

How to run me

Message me with the skill and audience:

Build a course on AI Literacy for non-technical marketing staff.

I draft each step and file a self-check with evidence; QA independently audits it against SKILL.md. On PASS I advance; on FAIL I get the failed criteria back (max 2 retries), then escalate to you.

Each step logs a verdict — PASS or FAIL — plus the criteria checked and retry count. That is your proof the gate has teeth.

After Step 4 I stop and show you the proposed architecture. Approve to continue, or ask for changes. You also designate the core module at Step 4.5.

On approval I run Step 5→8 (scripts, assessments, hand-off package). Everything lands in output/, with a run-log.md for your before/after numbers.
